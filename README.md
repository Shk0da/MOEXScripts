# Arb-Bot: Сканеры инвестиционных возможностей

Набор скриптов на Kotlin для анализа финансовых инструментов на Московской бирже.

## Установка

### Требования

- **Java JDK 11+** (рекомендуется 21) — для запуска Kotlin-скриптов
- **Kotlin 1.9+** — компилятор для файлов `.kts`
- Доступ к интернету

### Установка Kotlin

**Windows** — скачайте [kotlin-compiler](https://github.com/JetBrains/kotlin/releases/latest), распакуйте в `C:\Tools\kotlin\` и добавьте `C:\Tools\kotlin\kotlinc\bin` в переменную PATH пользователя.

**macOS / Linux** — через [SDKMAN](https://sdkman.io/):

```bash
sdk install kotlin
```

Проверка:

```bash
kotlin -version
java -version
```

После изменения PATH перезапустите терминал.

### Настройка API

Создайте файл `application.properties` в корне проекта:

```properties
tcs.apiKey=your-t-invest-token
finam.apiKey=your-finam-api-secret
```

Токен T-Invest можно получить в [личном кабинете T-Invest API](https://www.tbank.ru/invest/settings/api/).

## 1. Арбитраж акций и фьючерсов (arbitrage-scanner.kts)

Скрипт для поиска арбитражных возможностей между акциями и фьючерсами.

## Принцип работы

Скрипт сканирует фьючерсы на акции и находит пары с положительным контанго (когда фьючерс торгуется дороже акции). Стратегия:
1. Купить акцию
2. Продать фьючерс (шорт)
3. Держать до экспирации
4. Получить безрисковую прибыль (контанго)

### Учитываемые издержки:
- **Комиссия брокера** — 0.05% за сделку (4 операции: покупка акции, продажа фьючерса, закрытие позиции)
- **Стоимость ГО** — 16% годовых на сумму гарантийного обеспечения
- **Овернайт** — плата за перенос шорт-позиции через ночь (прогрессивная шкала)

## Требования

- Kotlin 1.9+ (для запуска скриптов .kts)
- Доступ к интернету (данные загружаются с MOEX ISS API)
- T-Invest API токен (опционально, для получения цен из стакана)

## Настройка T-Invest API

Для получения более точных цен из стакана создайте файл `application.properties`:

```properties
tcs.accountId=your-account-id
```

Токен можно получить в личном кабинете T-Invest API.

Если токен не указан, скрипт будет использовать цены MOEX.

## Запуск

```bash
# Базовый запуск со стандартными настройками
kotlin arbitrage-scanner.kts

# Свой капитал (по умолчанию 100 000 руб)
kotlin arbitrage-scanner.kts --capital 500000

# Фиксированный размер позиции (вместо распределения капитала)
kotlin arbitrage-scanner.kts --position-size 50000

# Своя комиссия брокера в % (по умолчанию 0.05%)
kotlin arbitrage-scanner.kts --commission 0.1

# Конкретные тикеры через запятую
kotlin arbitrage-scanner.kts --tickers SBER,GAZP,ROSN

# Отключить плату за овернайт
kotlin arbitrage-scanner.kts --no-overnight

# Отключить использование стакана T-Invest API (использовать MOEX)
kotlin arbitrage-scanner.kts --no-orderbook

# Комбинация параметров
kotlin arbitrage-scanner.kts --capital 1000000 --position-size 50000 --commission 0.03 --tickers SBER,TATN
```

## Вывод

Скрипт выводит:
- Цену акции и фьючерса
- Контанго в % и рублях
- Необходимое ГО (гарантийное обеспечение)
- Комиссии брокера
- Чистую прибыль и доходность
- Годовую доходность

## Интерпретация результатов

**Контанго** — разница между ценой фьючерса и акции. Положительное контанго означает что фьючерс дороже акции.

**Доходность** — чистая прибыль после вычета комиссий и стоимости финансирования ГО.

**Годовая доходность** — доходность пересчитанная на годовой период.

### Риски по сроку до экспирации:
- **Критический** (≤7 дней) — высокая годовая доходность, но мало времени
- **Низкий** (≤30 дней) — оптимальный баланс
- **Средний** (≤90 дней) — умеренный риск
- **Высокий** (>90 дней) — больше риск изменения условий

## Пример вывода

```
#1 GAZP | GZM7 | Высокий | 366дн.
   АКЦИЯ:  4 контр. x 100 акц. @ 111,75 = 44 700 руб.
   ФЬЮЧЕРС: 4 контр. @ 131 руб. ГО: 2 388 руб.
```

## Ограничения

- Не все фьючерсы имеют стандартный масштаб цены (MAGN, AFKS исключены)
- Данные загружаются в реальном времени с MOEX

## 2. Золотой арбитраж (gold-arbitrage.kts)

Скрипт для поиска арбитражных возможностей между мировым золотом (Forex XAU/USD) и российским золотом (MOEX GLDRUB).

### Принцип работы

Скрипт рассчитывает арбитражное соотношение по формуле:

```
Соотношение = (XAU/USD × USD/RUB) / GLDRUB
```

Где:
- **XAU/USD** — цена золота в долларах за тройскую унцию (Multi-Source fallback)
- **USD/RUB** — курс доллара к рублю (T-Invest / MOEX / CurrencyAPI fallback)
- **GLDRUB** — цена золота в рублях за грамм (T-Invest / MOEX IMB7 fallback)

**Физический смысл**: сколько грамм российского золота можно купить на вырученные от продажи 1 унции мирового золота деньги.

**Справедливое значение**: ~31.1035 (количество грамм в тройской унции)

### Торговые сигналы

| Уровень | Значение | Сигнал | Действие |
|---------|----------|--------|----------|
| Верхняя граница | ~32.2 | ВЕРХ | Шорт XAU/USD + Лонг GLDRUB |
| Нижняя граница | ~30.0 | НИЗ | Лонг XAU/USD + Шорт GLDRUB |
| Среднее | ~31.5 | Нейтрально | Возврат к среднему, закрытие позиций |

### Архитектура источников

Скрипт использует multi-source fallback цепочку:

**XAU/USD** (порядок обхода — до первого рабочего):
1. GoldAPI — GET /price/XAU (цена, 0.03% спрэд)
2. CurrencyAPI — CDN JSON xau.json (цена, 0.03% спрэд)
3. Finam — CSV экспор xauusd (close, 0.03% спрэд)
4. Alfa Forex — парсинг HTML страницы

**USD/RUB**:
1. T-Invest API — USD000UTSTOM (имя)
2. MOEX ISS — currency/selt (PREVSETTLEPRICE)
3. CurrencyAPI — rub.json обратный курс (1 / USD-per-RUB)

**GLDRUBF**:
1. T-Invest API — GLDRUBF (имя)
2. MOEX ISS — futures/forts (LAST)

### Учитываемые издержки

- **Комиссия брокера** — 0.05% за сделку (4 операции: покупка + продажа по каждой ноге)
- **Спред форекс** — ~0.03% (реальный, считается по bid/ask)
- **Свопы overnight** — не учитываются, важно для долгосрочных позиций

### Настройка T-Invest API

Токен используется из файла `application.properties`:

```properties
tcs.apiKey=your-t-invest-token
```

Если токен не указан, скрипт использует MOEX и бесплатные API.

### Запуск

```bash
# Базовый запуск (автоматическая загрузка цен из API)
kotlin gold-arbitrage.kts

# Ручной ввод цен (если API недоступен)
kotlin gold-arbitrage.kts --xau-bid 3300 --xau-ask 3302 --gld-rub 10000 --usd-rub 90

# Показать справку
kotlin gold-arbitrage.kts --help
```

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--xau-bid` | XAU/USD bid (вручную) | auto |
| `--xau-ask` | XAU/USD ask (вручную) | auto |
| `--gld-rub` | GLDRUB руб/г (вручную) | auto |
| `--usd-rub` | USD/RUB (вручную) | auto |
| `--fair-value` | Справедливое значение | 31.1035 |
| `--upper` | Верхняя граница входа | 32.2 |
| `--lower` | Нижняя граница входа | 30.0 |
| `--mean` | Цель возврата к среднему | 31.5 |
| `--commission` | Комиссия брокера в % | 0.05 |
| `--notional` | Номинал сделки в рублях | 1 000 000 |

### Вывод

Скрипт показывает:
- Котировки XAU/USD (bid/ask/mid, спрэд), GLDRUB, USD/RUB
- Рассчитанное арбитражное соотношение (Bid / Ask / Средняя)
- Отклонение от справедливого значения в ₽ и %
- Сигнал: ВЕРХ / НИЗ / Нейтрально
- Уровень риска: Низкий / Средний / Высокий
- Расчёт для каждой ноги: вход, цель, движение, валовая, затраты, чистая
- Действие: Шорт XAU/USD + Лонг GLDRUB / Лонг XAU/USD + Шорт GLDRUB

### Пример вывода

### Интерпретация результатов

**Соотношение > 32.2 (ВЕРХ)**: Мировое золото торгуется с премией к российскому. Выгодно продавать мировое золото (Forex) и покупать российское (MOEX).

**Соотношение < 30.0 (НИЗ)**: Мировое золото торгуется с дисконтом. Выгодно покупать мировое золото и продавать российское.

**Цель закрытия**: При возврате соотношения к среднему значению (~31.5) закрывайте обе позиции с прибылью.

### Риски

1. **Валютный риск** — GLDRUB номинирован в рублях, XAU/USD в долларах
2. **Свопы overnight** — удержание позиции дольше дня требует уплаты свопов
3. **Ликвидность** — GLDRUB может иметь низкую ликвидность на MOEX
4. **Размер контракта** — на Forex 1 лот = 100 унций, на MOEX золото торгуется в граммах
5. **Время работы бирж** — Forex работает 24/5, MOEX только в торговые сессии

### Хеджирование

Для нейтральной позиции правильно рассчитывайте объем:

```
1 лот XAU/USD (100 унций) = 3110.35 грамм GLDRUB
```

Пример:
- Шорт 1 лот XAU/USD → Лонг 3110 грамм GLDRUB

### Ограничения

- T-Invest Invest API — gRPC-протокол, JSON-запросы возвращают null; используется fallback на MOEX
- MOEX данные могут задерживаться на 15 минут
- Не учитываются свопы за перенос позиций overnight
- Не является инвестиционной рекомендацией

## 3. Сканер облигаций (bond-scanner.kts)

Скрипт для поиска топ-N облигаций с фиксированным купоном, заданной периодичностью и без амортизации. По умолчанию исключаются флоатеры и бумаги только для квалифицированных инвесторов.

### Принцип работы

Скрипт анализирует все доступные облигации и отбирает лучшие по доходности к погашению (YTM) с учетом:
- Частоты купонных выплат (ежемесячно = 12 раз в год)
- Отсутствия амортизации
- Исключения флоатеров (плавающий/переменный купон)
- Исключения бумаг только для квалифицированных инвесторов
- Уровня риска эмитента из T-Invest API (`riskLevel`)
- Фундаментальных показателей эмитента для топ-N (`GetAssetFundamentals`)
- Срока до погашения

### Настройка T-Invest API

Токен обязателен — данные загружаются из T-Invest API:

```properties
tcs.apiKey=your-t-invest-token
```

### Запуск

```bash
# Базовый запуск
kotlin bond-scanner.kts

# С параметрами
kotlin bond-scanner.kts --min-days 180 --max-days 730 --count 15

# Включить флоатеры и бумаги для квал. инвесторов
kotlin bond-scanner.kts --allow-floating --allow-qual
kotlin bond-scanner.kts -f -q

# Только надежные (AAA)
kotlin bond-scanner.kts --min-risk AAA

# Показать справку
kotlin bond-scanner.kts --help
```

### Параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--yield <value>` | Целевая доходность, % | 16.0 |
| `--coupon-freq <value>` | Частота купона в год | 12 |
| `--allow-amortization` | Разрешить амортизацию | false |
| `--allow-floating`, `-f` | Разрешить флоатеры | false |
| `--allow-qual`, `-q` | Разрешить бумаги для квал. инвесторов | false |
| `--min-risk <level>` | Макс. риск T-Invest (`AAA`, `MODERATE`, …) | `RISK_LEVEL_MODERATE` |
| `--count <value>` | Кол-во облигаций в топе | 10 |
| `--capital <value>` | Капитал для инвестиций | 100000 |
| `--min-days <value>` | Мин. дней до погашения | 90 |
| `--max-days <value>` | Макс. дней до погашения | 730 |

### Фильтры для неквалифицированных инвесторов и фиксированного купона

Скрипт автоматически исключает неподходящие бумаги (можно отключить флагами `--allow-floating` / `-f` и `--allow-qual` / `-q`):

| Поле API | Значение | Действие |
|----------|----------|----------|
| `forQualInvestorFlag` | `false` | Доступно неквалифицированным — **включается** |
| `forQualInvestorFlag` | `true` | Только для квал. инвесторов — **исключается** |
| `floatingCouponFlag` | `false` | Фиксированный купон — **включается** |
| `floatingCouponFlag` | `true` | Плавающий купон — **исключается** |

Дополнительно при расчёте купонов из API проверяется тип купона: учитываются только `COUPON_TYPE_FIX`, `COUPON_TYPE_FIXED`, `COUPON_TYPE_CONSTANT`; облигации с купонами `COUPON_TYPE_FLOATING` / `COUPON_TYPE_VARIABLE` отбрасываются (на случай неточного `floatingCouponFlag` в API).

При `--allow-floating` топ строится по **размеру ближайшего купона** (руб.), а не по YTM — для плавающих бумаг доходность к погашению менее предсказуема.

Пример статистики при запуске:

```
  Найдено: 1549
  Доступно неквалифицированным: 1200, без флоатеров: 980
  После фильтрации: 45
```

### Уровни риска (T-Invest)

Фильтрация и отображение риска берутся из поля `riskLevel` API T-Invest. **По умолчанию** в поиск попадают **надежные и средние** облигации (`RISK_LEVEL_LOW` + `RISK_LEVEL_MODERATE` = **AAA, AA-BB**).

| Уровень T-Invest | Рейтинг | Описание |
|------------------|---------|----------|
| `RISK_LEVEL_LOW` / `AAA` | AAA | Надежные эмитенты |
| `RISK_LEVEL_MODERATE` | AA-BB | Средняя надежность |
| `RISK_LEVEL_HIGH` | B-CCC | Повышенный риск |
| `RISK_LEVEL_VERY_HIGH` | D | Очень высокий риск |

Параметр `--min-risk` задаёт **максимально допустимый** уровень риска:
- по умолчанию `MODERATE` — надежные + средние
- `--min-risk AAA` — только надежные
- `--min-risk HIGH` — надежные, средние и повышенный риск

Купоны загружаются из T-Invest API (`GetBondCoupons`).

### Фундаментальный анализ эмитентов

Для облигаций из топа скрипт загружает финансовые показатели эмитента через `GetAssetFundamentals`:
- **D/E** (долг/капитал), **NetDebt/EBITDA**, **текущая ликвидность**
- **ROE**, **ROIC**, **чистая маржа**, **рост выручки за 5 лет**

Оценка состояния эмитента (колонка **Фин.**) — шкала **1–5**:

| Оценка | Состояние |
|--------|-----------|
| 1 | плохо |
| 2 | слабо |
| 3 | средне |
| 4 | хорошо |
| 5 | отлично |

Если по `assetUid` облигации данных нет, эмитент ищется среди акций (`GetAssets`). Для эмитентов без публичной отчётности — `н/д`.

### Вывод

Скрипт показывает:
- Тикер и название облигации
- Текущую цену (% от номинала)
- Размер купона на 1 облигацию
- Текущую доходность
- Доходность к погашению (YTM)
- Дней до погашения
- Уровень риска T-Invest (AAA, AA-BB, …)
- Оценка фин. состояния эмитента (1–5)
- Сектор эмитента
- Пример портфеля с расчетом купонных выплат

### Пример вывода

```
#    | Тикер        | Название                  |  Цена  |  Купон   | Купон% |  YTM%  | Дней  |    Риск    |  Фин.  |    Сектор
1    | RU000A10AHE5 | Новые Технологии 001Р-02  | 106,34% |  22,60 руб |  25,5% |  19,3% |   183  | AAA        | 4 хор. | materials
```

### Ограничения

- API T-Invest имеет rate limit (429 Too Many Requests)
- Некоторые облигации могут не иметь цен в стакане
- Купоны в иностранной валюте исключаются из расчета
- Облигации с неизвестным `riskLevel` не проходят фильтр
- Не является инвестиционной рекомендацией

## 4. Сканер портфеля (portfolio-scanner.kts)

Скрипт для сводного анализа портфеля из Finam + T-Invest: загружает все счета обоих брокеров, объединяет позиции, загружает купоны и дивиденды, строит график выплат.

### Принцип работы

1. **Авторизация в Finam API** — POST `/v1/sessions` с `secret` → JWT
2. **Получение счетов Finam** — POST `/v1/sessions/details` → массив `account_ids`
3. **Загрузка портфеля Finam** — GET `/v1/accounts/{id}` → позиции, капитал, кэш
4. **Получение счетов T-Invest** — `UsersService/GetAccounts` → 8 типов счетов (TINKOFF, IIS, INVEST_BOX, INVEST_FUND, DFA)
5. **Загрузка справочника облигаций** — `InstrumentsService/Bonds` (1549 шт.)
6. **Загрузка портфеля T-Invest** — `OperationsService/GetPortfolio` для каждого счета
7. **Поиск имён инструментов**:
   - Облигации: поиск по ticker/figi в справочнике
   - Акции/ETF: `FindInstrument` с фильтром по типу и classCode
8. **Слияние позиций** — объединение по figi, пересчёт средневзвешенной цены
9. **Загрузка купонов** — `InstrumentsService/GetBondCoupons` по figi
10. **Загрузка дивидендов** — `InstrumentsService/GetDividends` по figi/instrumentId
11. **Вывод сводной таблицы** — все позиции с источником (F / T / F+T)
12. **График выплат** — купоны + дивиденды по месяцам с ASCII bar chart и доходностью
13. **Детализация по счетам** — капитал и кэш каждого счёта

### Требования

- Kotlin 1.9+
- Токен T-Invest API
- Finam API ключ (секрет сессии)

### Настройка

В `application.properties`:

```properties
tcs.apiKey=your-t-invest-token
finam.apiKey=your-finam-api-secret
```

### Запуск

```bash
kotlin portfolio-scanner.kts
```

### Особенности интеграций

#### Finam API
- Аутентификация: `POST /v1/sessions` с `{"secret": "..."}` → JWT
- Список счетов: `POST /v1/sessions/details` → поле `account_ids`
- Получение портфеля: `GET /v1/accounts/{account_id}` (equity, cash, positions)
- Срок жизни JWT: ~15 минут
- Цена облигаций (average_price, current_price) — в % от номинала

#### T-Invest API
- **GetPortfolio**: `{"accountId": "...", "currency": "RUB"}` → позиции с figi, ticker, classCode, ценами
- **GetBondCoupons**: параметр `figi` (не `instrumentId`)
- **GetDividends**: `{"instrumentId": "<figi>"}` (без from/to), ответ `{"dividends": [{"dividendNet": {...}, "paymentDate": "..."}]}`
- **FindInstrument**: поиск имени по тикеру с фильтром `instrumentKind` + classCode
- **GetWithdrawLimits**: кэш по счёту (недоступен для INVEST_BOX, INVEST_FUND, DFA)
- Rate limit: 429 Too Many Requests (retry 3 попытки + Thread.sleep(150ms))

#### SSL
- macOS/Java не доверяет российским УЦ
- Используется `createTrustAllSSLContext()` — trust-all SSLContext

### Вывод

Скрипт показывает:
- Все позиции из обоих брокеров в единой таблице (тип, тикер, название, кол-во, средняя, цена, стоимость, доля, купон/мес, доходность, погашение, источник)
- Источник: `F` (Finam), `T` (T-Invest), `F+T` (в обоих)
- Ожидаемые купоны и дивиденды по месяцам с ASCII bar chart
- Средняя доходность: по всему портфелю и только по платящим бумагам

## 5. Сканер фундаментальных показателей (fundamentals-scanner.kts)

Скрипт для поиска недооценённых компаний на основе фундаментальных мультипликаторов и прогнозов аналитиков.

### Принцип работы

Скрипт использует три эндпоинта T-Invest API:

1. **`InstrumentsService/Shares`** — получение списка всех торгуемых акций (1980 шт.)
2. **`InstrumentsService/GetAssets`** — получение активов и построение маппинга инструмент → актив (1349 активов)
3. **`InstrumentsService/GetAssetFundamentals`** — фундаментальные показатели по каждому активу:
   - P/E (TTM), P/B (TTM), P/S (TTM), EV/EBITDA
   - ROE, ROA, ROIC, Net Margin, Current Ratio
   - Дивидендная доходность (forward + daily TTM)
   - 5-летний рост выручки, Beta, Free Float
   - Рыночная капитализация, долг/EBITDA, D/E
4. **`InstrumentsService/GetForecastBy`** — прогнозы инвестдомов для топ-10 бумаг

### Методология оценки

Бумаги ранжируются по взвешенной сумме баллов:

| Мультипликатор | Вес | Диапазон (макс → мин баллы) |
|----------------|-----|----------------------------|
| P/E | ×3 | ≤5 → 100, ≤10 → 85, ≤15 → 65, ≤20 → 45, ≤30 → 20 |
| P/B | ×2 | ≤0.5 → 100, ≤0.7 → 90, ≤1 → 75, ≤1.5 → 55, ≤2 → 35 |
| EV/EBITDA | ×2 | ≤3 → 100, ≤5 → 80, ≤8 → 60, ≤12 → 40, ≤20 → 20 |
| Дивиденды | ×1 | ≥15% → 100, ≥10% → 85, ≥6% → 65, ≥3% → 45 |
| ROE | ×1 | ≥30% → 100, ≥20% → 80, ≥15% → 60, ≥10% → 40 |
| D/E | ×1 | ≤10% → 100, ≤30% → 80, ≤50% → 60, ≤100% → 40 |
| Рост выручки (5л) | ×1 | ≥20% → 100, ≥10% → 80, ≥5% → 60 |

Максимум — 100 баллов. Фильтр: капитализация ≥ 1 млрд ₽, P/E от 0 до 50.

### Требования

- Kotlin 1.9+
- Токен T-Invest API (`tcs.apiKey`)

### Настройка

В `application.properties`:

```properties
tcs.apiKey=your-t-invest-token
```

### Запуск

```bash

```

### Вывод

Скрипт выводит:

1. **Таблица топ-10 бумаг** — тикер, название, сектор, P/E, P/B, EV/EBITDA, D/E, ROE, ROIC, див.доходность, рост выручки, оценка
2. **Прогнозы аналитиков** для каждой из топ-10 бумаг:
   - Консенсус-цена и диапазон (min–max)
   - Текущая цена и потенциал роста
   - Список инвестдомов с целевыми ценами и рекомендациями (Т-Инвестиции, Альфа-Банк, Сбер, Газпромбанк, Финам, БКС и др.)
3. **Итоговый рейтинг** — детальная разбивка по каждому мультипликатору

### Фильтр для неквалифицированных инвесторов

Скрипт автоматически фильтрует бумаги, доступные только квалифицированным инвесторам.
Используется поле `forQualInvestorFlag` из API T-Invest:
- `forQualInvestorFlag: false` — доступно неквалифицированным инвесторам
- `forQualInvestorFlag: true` — только для квалифицированных инвесторов (исключается из результатов)

При запуске показывается статистика:
```

```

## 6. Анализатор новостей (news-analyzer.kts)

Скрипт для получения актуальных новостей из T-Bank Invest API, обогащения их фундаментальными показателями и прогнозами аналитиков, и анализа с помощью локальной LLM (Ollama).

### Принцип работы

1. **Загрузка новостей** — `InstrumentsService/News` с курсорной пагинацией (100 новостей на страницу)
2. **Фильтрация** — оставляются только новости за текущую дату (поле `ts`)
3. **Загрузка инструментов** — `InstrumentsService/GetAssets` (instrumentType: INSTRUMENT_TYPE_SHARE) → маппинг instrumentUid → (assetUid, ticker, figi)
4. **Фундаментальные показатели** — `InstrumentsService/GetAssetFundamentals` по assetUid:
   - P/E (TTM), P/B (TTM), EV/EBITDA, ROE, чистая рентабельность, Beta
   - Рыночная капитализация, дивидендная доходность, долг/EBITDA, D/E
   - 5-летний рост выручки
5. **Прогнозы инвестдомов** — `InstrumentsService/GetForecastBy` по FIGI:
   - Консенсус-цена, диапазон (min–max), потенциал роста
   - Список инвестдомов с целевыми ценами и рекомендациями (Покупать/Держать/Продавать)
6. **Вывод списка** — заголовок, текст, фундаментальные показатели и прогнозы по каждому инструменту
7. **Анализ LLM** — до 50 новостей с контекстом мультипликаторов и прогнозов отправляются в локальную Ollama

### Требования

- Kotlin 1.9+
- Токен T-Invest API (`tcs.apiKey`)
- [Ollama](https://ollama.ai) с моделью `0xroyce/plutus` (или другая)

### Детали реализации

#### Маппинг инструментов

Цепочка разрешения instrumentUid → данные:
```
instrumentUid (из новости)
  → GetAssets: instrumentUid → assetUid + ticker + figi
    → GetAssetFundamentals(assetUid) → P/E, P/B, ROE, Cap, ...
    → GetForecastBy(figi) → консенсус, целевые цены, рекомендации
```

Ключевые классы:
- `InstrumentAssetInfo` — маппинг instrumentUid → (assetUid, ticker, figi)
- `FundamentalData` — мультипликаторы и метрики
- `ForecastData` / `ForecastTarget` — прогнозы инвестдомов

#### Обработка пустых ticker в GetAssetFundamentals

API `GetAssetFundamentals` возвращает объекты без поля `ticker`. Скрипт
восстанавливает тикер через обратный маппинг `assetUid → ticker`,
построенный на этапе загрузки GetAssets.

### Анализ LLM

Скрипт формирует промпт на русском языке и отправляет в Ollama. Для каждой новости LLM получает:

- **Тематику** — макроэкономика, отрасль, компания, регуляторика
- **Сентимент** — позитивный / негативный / нейтральный
- **Затронутые сектора и инструменты** с учётом мультипликаторов (P/E, P/B, ROE, Div, EV/EBITDA)
- **Прогнозы аналитиков** — консенсус-рекомендация, целевая цена, потенциал

В конце выдаётся общий вывод: формирующиеся тренды, на какие инструменты
стоит обратить внимание с учётом их оценки и прогнозов.

| `InstrumentsService/GetAssets` | Получение активов (акции) с instrumentUid → assetUid + figi |
| `InstrumentsService/GetAssetFundamentals` | Фундаментальные показатели по assetUid |
| `InstrumentsService/GetForecastBy` | Прогнозы аналитиков по FIGI |

### Ограничения

- LLM-анализ требует запущенного локального сервера Ollama
- При большом количестве новостей (>1000) загрузка может занять до 30 секунд
- Фундаментальные данные доступны для ~730 из 1350 активов
- Прогнозы аналитиков — только для ограниченного числа наиболее ликвидных бумаг
- Модель `0xroyce/plutus` может давать неточные выводы — не является инвестиционной рекомендацией
- API T-Invest имеет rate limit (429), но скрипт не делает retry для News

## 7. Сканер дивидендов (dividend-scanner.kts)

Скрипт для поиска акций с лучшей дивидендной историей, растущими дивидендами и прогнозом выплат на 5 лет.

### Принцип работы

Скрипт анализирует все доступные акции ММВБ и отбирает лучшие по дивидендной истории:

1. **Загрузка списка акций** — `InstrumentsService/Shares` (фильтр: неквалифицированные инвесторы, рублевые акции)
2. **Загрузка дивидендов** — `InstrumentsService/GetDividends` за последние 7 лет
3. **Агрегация по годам** — суммирование дивидендов за каждый год
4. **Расчёт метрик**:
   - Конsecutive years — количество подряд лет выплат
   - CAGR — среднегодовой темп роста дивидендов
   - Текущая доходность — дивиденд за последний год / текущая цена
   - Прогноз на 5 лет — экстраполяция на основе исторического роста
5. **Ранжирование** — сначала по consecutive years, потом по CAGR, потом по доходности

### Методология оценки

Акции ранжируются по трём критериям (приоритет уменьшения):

| Критерий | Описание |
|----------|----------|
| Consecutive Years | Количество подряд лет с выплатами (макс. 8 лет) |
| CAGR | Среднегодовой темп роста дивидендов (%) |
| Current Yield | Текущая дивидендная доходность (%) |

### Прогноз на 5 лет

Прогноз рассчитывается на основе CAGR:

```
Прогноз = Текущий_дивиденд × (1 + CAGR)^n
```

где n — номер года вперед (1..5).

### Запуск

```bash
# Базовый запуск
kotlin dividend-scanner.kts
```

### Вывод

Скрипт показывает:
- Топ-10 акций с лучшей дивидендной историей
- Текущую цену и доходность
- Рост дивидендов (CAGR) в %
- Количество подряд лет выплат
- Прогноз выплат на 5 лет
- Историю выплат по годам
- Детальную информацию по каждой акции из топ-10
